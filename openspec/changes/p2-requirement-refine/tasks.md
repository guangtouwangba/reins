- [ ] **Task 5: Implement requirement refiner**
  - Description: Create `src/pipeline/requirement-refiner.ts`. Export `refineRequirements(task, context, constraints, opts): Promise<string>` (returns spec ID). Implement the flow: (1) Call `generateQuestions(task, context, constraints)`. (2) Print inferred facts to stdout as "Based on your project, I already know:". (3) Prompt user for each blocking question, then each important question (using readline or `@inquirer/prompts`). (4) Apply defaults for optional questions. (5) Call `specGenerator.generate()` (Task 6) with answers. (6) Print generated spec to stdout. (7) Prompt user for confirmation: `[y/n/edit]`. On `y` → write spec.md and return spec ID. On `n` → abort. On `edit` → open `$EDITOR` on temp file, re-read, re-display, re-confirm. (8) In non-interactive mode (`opts.noInput`): skip prompts, use all defaults, auto-confirm. (9) Create spec directory via `specs.ts`, write spec.md, update index.yaml with status `confirmed`.
  - Files: `src/pipeline/requirement-refiner.ts`
  - Tests: Unit test in non-interactive mode → assert spec generated without prompts, all defaults applied. Unit test that inferred facts are displayed (capture stdout). Unit test that user answering `n` aborts without writing files. Unit test that spec.md is written to correct path. Unit test that index.yaml is updated with status `confirmed`.
  - Done when: Interactive flow works in TTY; non-interactive mode skips prompts; edit flow works; spec persisted; index updated
